# reelcast — プロジェクト方針書（PLAN）

> 個人開発。YouTube / Instagram 向けに、分析 → マーケ企画 → 制作 → 投稿 → 運用 を AI で一気通貫に行うシステム。
> 本書は初期方針策定セッション（2026-05-30）で対話的に確定した決定の記録。実装方針の単一の参照点とする。

---

## 0. 前提となる制約（すべての決定の土台）

| 制約 | 内容 | 含意 |
|---|---|---|
| 最優先ゴール | **収益化** | 収益化条件という「門」が常に効く |
| 投下時間 | **〜5時間/週** | **自動化レベルを高く保つ**ことが生存条件 |
| 初期資産 | **ほぼゼロから** | 数ヶ月の助走を前提に設計 |
| 予算 | **無料中心（〜数百円）。例外は音楽ツールのみ** | 音楽の商用権だけは有料で確保 |
| 環境 | **ローカル Mac 中心** | 重い生成・合成はローカルで完結 |

## A. 戦略・ゴール（確定）

| 項目 | 決定 |
|---|---|
| 最優先ゴール | 収益化 |
| コンテンツ土台 | **AI機能性音楽**（作業用 / 睡眠 / lo-fi / 環境音）の映像付き長尺 |
| 演者性 | **完全フェイスレス + テキスト人格のみ**（バーチャルレーベル名義） |
| 収益の土台 | **YouTube 広告収益**（長尺・高視聴維持・エバーグリーン） |
| 第二の柱（将来） | 素材 / ロイヤリティフリー音源のライセンス販売（音源資産の転用＝"ツルハシ"側） |
| 主戦場の役割分担 | **YouTube＝収益本体（長尺）** / **Instagram＝認知・集客（ショート切り出し）** |

### 戦略の根拠（AIエージェント時代の防御性）
- 情報アービトラージ型（比較・要約・ベスト10）のアフィリエイトは **AIエージェントに中抜きされる構造的衰退**。当初検討したアフィリエイト主軸はこの理由で土台から外した。
- 音楽は **「エンタメ・時間消費そのもの」** であり、AIエージェントに代替されにくい。フェイスレス＋テキスト人格とも整合。

### ⚠️ 戦略の隘路：登録者数
- YouTube 広告収益には YPP 参加が必要（⚠️**要確認**：概ね登録1,000人＋年間総再生4,000時間）。
- 機能性音楽は **総再生時間は貯まりやすい**（長尺を流しっぱなし）が、**「流すが登録しない」視聴行動で登録率が構造的に低い**。
- → 成否は **「登録したくなる仕掛け（チャンネルブランド・シリーズ化・世界観）」** の設計に懸かる。これが B の中心課題。

### KPI と最初の3ヶ月（目標水準：手堅め）
- **追うKPI**：①公開本数 ②総再生時間 ③平均視聴維持 ④**登録者数（隘路）** ⑤プレイリスト保存
- 収益発生は現実的に YPP 通過後（≒3ヶ月以降）になる前提。
- **Month 1**：半自動パイプライン MVP／チャンネル開設・世界観確定／長尺 5〜10 本／IG 切り出し運用開始
- **Month 2**：公開ペース確立（週2〜3本）／勝ちフォーマット特定／プレイリスト戦略
- **Month 3**：総再生時間を積み上げ YPP 条件への軌道に乗せる／登録施策の検証／条件到達なら YPP 申請

## B. コンテンツ方針（確定）

| 項目 | 決定 |
|---|---|
| 長尺の型 | **テーマ別**（45分〜1時間、「雨の夜カフェ」「集中モード」等のシリーズ） |
| 世界観 | **キャラクター型**（固定キャラを看板に） |
| キャラ実装方式 | **固定キャラ資産方式**（一度作り込み全動画で使い回す。変わるのは音楽と背景のみ＝Lofi Girl 方式） |
| 投稿頻度 | **長尺 週2 ＋ ショート 週3〜4**（ショートは長尺の切り出し） |
| 自動化レベル | **生成・編集は自動／公開は人間承認**（週次バッチで品質・規約ゲート） |

### ⚠️ キャラクター型の設計指針
- AI動画生成は「同じキャラを毎回一貫させる」のが苦手。**都度生成は破綻する**。
- → キャラは **固定アセット**として作り込み、背景と音楽のみ差し替える。一貫性と量産を両立する唯一の現実解。

## C. 分析・マーケ（確定）

> 前提転換：機能性音楽は **トレンド追いではなく、エバーグリーン需要を検索・関連・プレイリストで取りに行く SEO/発見性ゲーム**。

| 項目 | 決定 |
|---|---|
| 分析の主眼 | **需要 / SEO 起点**（検索・関連・プレイリストで需要網羅、タイトル/サムネ SEO 重視） |
| データソース | 公式 YouTube Data API（無料枠）＋ Google トレンド＋無料ツール。**スクレイピングは不採用（規約）** |
| 企画フロー | **AI がデータからテーマ候補を複数提案 → 人間が週次で選定** |
| Instagram 分析 | 初期は手動寄り（Graph API は Business アカウント＋制限のため） |

### ⚠️ データ取得の許容範囲（要確認・仕様変動）
- YouTube Data API v3：無料枠 1日10,000ユニット。検索系は消費が大きく、設計で節約が要る。
- スクレイピングは YouTube/Instagram とも ToS で原則禁止。公式 API か、公式 API 経由のサードパーティツールのみ。
- Instagram Graph API は Business/Creator アカウント必須・取得範囲限定。

## D. 制作パイプライン（確定）

### 工程分解
```
①テーマ選定(AI提案→人間選定)
   → ②音楽生成(有料AI・商用権あり / 複数トラックを45-60分に構成)
   → ③キャラ資産(初期に一度だけ作る固定アセット) + ④背景ビジュアル生成(テーマ別に差し替え)
   → ⑤映像合成・長尺化(キャラ+背景+音楽をループ合成)
   → ⑥サムネ生成 → ⑦メタデータ/SEO(タイトル・説明・章・タグ)
   → ⑧公開承認(人間ゲート/週次バッチ) → ⑨アップロード → ⑩ショート切り出し
```

| 工程 | 方針 |
|---|---|
| 音楽生成 | **Suno Pro（$10/月）＋ 人間アシスト**（公式APIなし／第三者APIは規約リスクで不採用）。商用権は継続・印税0%徴収。詳細・根拠は [`docs/TOOLS.md`](TOOLS.md) |
| 映像生成 | **無料/低コストの画像生成 AI で背景＋固定キャラは一度作る**。具体ツールは**キャラ設計と一緒に決定（保留）** |
| 合成・長尺化・編集 | **コード自動化**（ffmpeg / MoviePy 中心。長尺ループ合成は ffmpeg で軽量に） |
| 権利・開示 | **規約フル遵守** |

> **music 工程の設計**：Suno Pro は公式 API がないため、人間が Web で週次バッチ生成・DL し `data/assets/music/<video_id>/` に投入 → パイプラインが取り込み・検証して合成へ。生成自体は人間アシスト。音楽選定の人間キュレーションは **YouTube のAI音楽収益化要件（意味のある人間のキュレーション）にも合致**。将来フル自動化が必要なら Suno Premier（~$24・公式API）へ。

### ⚠️ 権利・開示の運用（要確認）
- 映像/音楽生成ツールの**商用利用可否**を必ず確認（無料ツールは非商用条件のことがある）。
- **AI生成コンテンツの開示**：YouTube の「改変・合成コンテンツ」開示要件に従う（隠す設計は採らない）。
- フォント・効果音・素材は**商用可ライセンスのみ**。ライセンス台帳で管理。
- **Content ID**：自作 AI 音楽は他者申立てを受けにくいが、自分の AI 音楽を Content ID 登録するのはリスク/不可の場合あり。慎重に。

## E. 投稿・運用（確定）

| 項目 | 決定 |
|---|---|
| YouTube 投稿 | **API で限定公開アップ → 人間が週次承認 → publishAt で予約公開** |
| Instagram 投稿 | **当面 手動 / 半自動**（API 制約・BAN リスク回避。YouTube 主軸の集客補助） |
| 改善ループ | **Analytics API で KPI 自動収集 → AI がテーマ提案に還流 → 人間が週次判断** |

### ⚠️ 投稿自動化の制約（要確認）
- YouTube Data API：アップロード＋予約公開（publishAt）可。アップロードは quota 消費が大きめ（週2本なら問題なし）。
- Instagram Content Publishing API：Business/Creator 必須、レート制限・仕様制約が強い。スパム的自動大量投稿は BAN 要因。
- **完全自動公開（人間ゲート無し）は採らない**（B の決定と一貫）。

> 注：収益モデルが広告収益のため、アフィリ用の「ハブ（まとめサイト/LP）」は必須ではない。将来 ④素材販売の柱を立てる時に再浮上する程度の位置づけ。

## F. 技術アーキテクチャ（確定）

| 項目 | 決定 |
|---|---|
| 実行環境 | **ローカル Mac 中心**（外部 API は有料音楽・無料枠 LLM/画像生成のみ） |
| オーケストレーション | **工程単位モジュール（疎結合）→ 初期は単純パイプライン → 将来 工程ごとにエージェント化** |
| 言語・スタック | **Python**（AI/メディア/エージェント基盤が厚い） |
| データストア | **SQLite（状態管理）＋ ローカルファイル（素材）** |

### 設計の勘所
- 本質は **「工程パイプライン＋状態管理＋人間承認キュー」**。
- 各動画は `企画→音楽→映像→合成→メタ→承認待ち→公開` の状態を遷移。
- 生成は失敗・やり直しが多いので、**各工程を冪等・再実行可能**に作る。
- エージェント分割の単位＝**パイプライン工程**（企画/音楽/映像/合成/メタ/公開）。

## G. 段階計画（確定）

### Phase 0 — 準備・基盤（〜2週間）
- [ ] YouTube ブランドアカウント開設、ポリシー確認 ⚠️
- [ ] AI 音楽ツールの有料プラン契約 ＋ **商用利用権の規約確認** ⚠️
- [ ] **固定キャラ資産の作成**（世界観の核・一度きり）
- [ ] シリーズ設計（最初のテーマ 3〜5 個）
- [ ] 開発環境（Python / SQLite / ffmpeg / リポジトリ初期化）
- [ ] API 設定（YouTube Data + Analytics API の OAuth、無料枠 LLM/画像生成キー）⚠️

### Phase 1 — MVP パイプライン（Month 1）
- [ ] 工程単位モジュールを最小実装（①〜⑩）
- [ ] 半自動で1本を最後まで通す＋**人間承認ゲート**（限定公開→予約公開）
- [ ] 長尺 5〜10 本公開、IG は切り出し手動
- [ ] SQLite による状態管理（冪等・再実行可能）

### Phase 2 — 運用安定化・改善ループ（Month 2〜3）
- [ ] 公開ペース確立（長尺 週2 ＋ ショート 週3〜4）
- [ ] Analytics API で KPI 自動収集 → AI 還流 → 週次レビュー
- [ ] 勝ちフォーマット特定、タイトル/サムネ SEO 最適化
- [ ] 総再生時間を積み上げ YPP 条件への軌道へ／登録施策の検証

### Phase 3 — 拡張（Month 4 以降）
- [ ] 工程のエージェント化（マルチエージェントオーケストレーション）
- [ ] Instagram 半自動化（Graph API 検討）
- [ ] 第二の柱（素材/ライセンス販売）の検討
- [ ] ②オリジナル楽曲MV など派生フォーマット
